# Dify 検索精度・最適化設定ガイド

RAG（Retrieval-Augmented Generation）システムにおいて、エラーコード（例: `ASC-3011`）や固有のコマンド名（例: `asctl`）を正確に引き当て、かつ無関係な情報によるハルシネーションを極限まで低減させるための、Dify ナレッジ設定の最適化ガイドです。

---

## 1. ハイブリッド検索 (Hybrid Search) の設定

### なぜ必要なのか？
* **ベクトル検索（セマンティック検索）の弱点**:
  「エラーコード ASC-3011」や「asctl sync」のような記号交じりの固有IDやコマンド名は、意味的類似度（ベクトル空間上の距離）では関連する他の文書に埋もれてしまいやすく、ヒット率が低下します。
* **キーワード検索（BM25）の強み**:
  文字の完全一致に非常に強いため、固有の識別子やコマンドを確実に引き当てることができます。

### 設定手順
1. Dify 管理画面にログインし、対象の **ナレッジ（Dataset）** を選択します。
2. ナレッジの「設定」または「リトリーバル設定（Retrieval Setting）」を開きます。
3. 検索設定（Search Setting）にて、デフォルトの「ベクトル検索（Vector Search）」から **「ハイブリッド検索（Hybrid Search）」** に変更します。

---

## 2. リランク（Reranking）の設定

ハイブリッド検索では、キーワード検索とベクトル検索の2つの結果が混在するため、そのままでは関連度の低いチャンクが上位に入ることがあります。これらを再評価し、最も関連度の高いチャンクを先頭にソートするために **リランカー (Reranker)** を導入します。

### 設定手順
1. ナレッジのリトリーバル設定画面にて、「Rerankモデル（Rerank Model）」を有効にします。
2. 以下のいずれかのリランカーを選択・設定します（モデルプロバイダー設定で API キーの追加が必要です）:
   * **Cohere Rerank**: 商用で非常に高精度なリランク性能を持ちます。
   * **BGE-Reranker-Large**: OSSとして公開されており、ローカル環境でも動かすことができる高精度なリランカーです。
3. リトリーバル設定パラメーターを以下のように調整します。

### 推奨パラメーター構成

| パラメーター | 推奨設定値 | 目的・効果 |
| :--- | :--- | :--- |
| **Top K** | `3` 〜 `5` | LLMに渡すコンテキスト数を最小限に絞り、情報の密度を高めてハルシネーションの混入を防ぐ。 |
| **最小信頼度スコア (Score Threshold)** | `0.5` 〜 `0.6` | 関連度が低い（類似度が一定以下の）チャンクを完全に切り捨てる。これにより、関係のないドキュメントに基づいてLLMが勝手な推測（ハルシネーション）をするのを防ぐ。 |
| **検索ウェイト（Rerank設定）** | `Custom Weight (ベクトル: 0.7 / キーワード: 0.3)` | 意味的な文脈（ベクトル）を重視しつつ、コマンド名やエラーコードの完全一致（キーワード）もブレンドするバランス設定。 |

---

## 3. チャンク（セグメンテーション）の最適化

ドキュメントの分割方法（チャンキング）によっても、検索の精度は大きく変わります。

### 設定手順
1. ナレッジへのドキュメント登録・更新時、「セグメンテーション設定（Chunking Settings）」にて **「カスタム（Custom）」** または **「親子セグメンテーション（Parent-Child Segment）」** を選択します。
2. 設定値の目安:
   * **最大チャンク長 (Segment Length)**: `500` 〜 `800` トークン
   * **オーバーラップ (Overlap)**: `50` 〜 `100` トークン（文脈の境界での情報欠落を防ぐため）
3. **親子セグメンテーション（Parent-Child Segment）の活用（推奨）**:
   * **子チャンク（200トークン程度）**: 細かく区切ることで、キーワードやベクトル検索時のマッチング感度を最大化します。
   * **親チャンク（800トークン程度）**: 実際にLLMへ提供するコンテキスト。ヒットした子チャンクの周辺文脈（親チャンク）ごとLLMに渡すことで、LLMは「途切れた文脈」によるハルシネーションを起こさずに回答を作成できます。
