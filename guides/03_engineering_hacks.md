
# Engineering Hacks: 技術的なこだわりと課題解決

本システムは、単なるツールの寄せ集めではなく、OSレベルの制限やAPIの仕様の壁を越えるための泥臭いエンジニアリングの上に成り立っています。

## 1. macOSにおける Fork 制限の完全バイパス

macOS (Darwin) 環境下では、FastAPI/Uvicornなどの親プロセスから `os.fork()` を用いてバックグラウンドデーモンを起動しようとすると、CoreFoundationのセキュリティ制約により `SIGABRT` で強制終了されます。
本プラットフォームの `deploy_listener.py` 等では、`subprocess.Popen` に `start_new_session=True` を指定することでプロセスグループを完全に切り離し、OSのシグナル連鎖を断ち切ることで安定したデーモン化を実現しています。

## 2. Webhook再起動時のデッドロック排除

GitHub Webhookをトリガーにシステム全体の再起動 (`ragy restart`) を走らせる際、「再起動命令を出したプロセス自身が再起動に巻き込まれて共倒れする」というデッドロックが発生しました。
これを防ぐため、Webhookレシーバーはリクエスト受理後に `os._exit(0)` でポートを即時解放して自死し、`ragy` スクリプト側で「ポートが空くまで最大10秒間ポーリング待機する」非同期ハンドオフ機構を実装しました。

## 3. Dify API仕様へのハック (multipart/form-data)

DifyのDocument APIへファイルを同期する際、メタデータを通常のフォームフィールドとして送信するとバリデーションエラーが発生します。
公式ドキュメントに明記されていないこの挙動に対し、ペイロードを単一の `data` キーに押し込み、その中で `indexing_technique: high_quality` 等の設定をJSONシリアライズして送信するというハックを施すことで、CLIからの完全な自律同期を成功させています。

## 4. エラー自己修復エージェント (`agent_healer.py`)

単純なログ監視にとどまらず、PythonのTracebackを検知するとAIエージェントが起動。ローカルソースコードのAST（抽象構文木）レベルでの問題特定から、修正パッチの適用、そしてGitHubへのPull Request作成までを人間の介入なしで完遂するSelf-Healingパイプラインを構築しています。
