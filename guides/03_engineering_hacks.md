
# Engineering Hacks: 技術的なこだわりと課題解決

本システムは、単なるツールの寄せ集めではなく、OSレベルの制限やAPIの仕様の壁を越えるための泥臭いエンジニアリングの上に成り立っています。

## 1. macOSにおける Fork 制限の完全バイパス

macOS (Darwin) 環境下では、FastAPI/Uvicornなどの親プロセスから `os.fork()` を用いてバックグラウンドデーモンを起動しようとすると、CoreFoundationのセキュリティ制約により `SIGABRT` で強制終了されます。
本プラットフォームの `deploy_listener.py` 等では、`subprocess.Popen` に `start_new_session=True` を指定することでプロセスグループを完全に切り離し、OSのシグナル連鎖を断ち切ることで安定したデーモン化を実現しています。

## 2. Webhook再起動時のデッドロック排除

GitHub Webhookをトリガーにシステム全体の再起動 (`ragy restart`) を走らせる際、「再起動命令を出したプロセス自身が再起動に巻き込まれて共倒れする」というデッドロックが発生しました。
これを防ぐため、Webhookレシーバーはリクエスト受理後に `os._exit(0)` でポートを即時解放して自死し、`ragy` スクリプト側で「ポートが空くまで最大10秒間ポーリング待機する」非同期ハンドオフ機構を実装しました。

## 3. Dify API仕様へのハック (multipart/form-data)

DifyのDocument APIへファイルを同期する際、メタデータを通常のフォームフィールドとして送信するとバリデーションエラーが発生します。
公式ドキュメントに明記されていないこの挙動に対し、ペイロードを単一の `data` キーに押し込み、その中で `indexing_technique: high_quality` 等の設定をJSONシリアライズして送信するというハックを施すことで、CLIからの完全な自律同期を成功させています。

## 4. エラー自己修復エージェント (`agent_healer.py`)

単純なログ監視にとどまらず、PythonのTracebackを検知するとAIエージェントが起動。ローカルソースコードのAST（抽象構文木）レベルでの問題特定から、修正パッチの適用、そしてGitHubへのPull Request作成までを人間の介入なしで完遂するSelf-Healingパイプラインを構築しています。

## 5. 「Lost in the Middle (中だるみ)」問題の克服 (コンテキスト最適化)

LLMが「長いコンテキストの最初と最後にある情報に最も注目し、中央にある情報を軽視する」というLost in the Middle問題に対処するため、Difyハイブリッド検索から取得した検索結果の並び替えアルゴリズムを開発。
最高スコアのドキュメントを先頭に、次に高いドキュメントを末尾に、低スコアのものを中間に隠すように再配置 (`[Rank 1, Rank 3, Rank 5, Rank 4, Rank 2]`) してプロンプトを構築します。このロジックは Python (MCP/CLI) と Rust (TUI) の双方に共通で堅牢に実装されています。

## 6. 標準画像ファイルのVision OCRマルチモーダル同期

`.pdf`等の文書のみならず、プロジェクトディレクトリに置かれた設計図、UML、画面キャプチャ画像 (`.png`, `.jpg`, `.jpeg`) をそのまま監視対象としてサポート。Pillowライブラリと LiteLLM Vision (Gemini 2.5 Flash / Gemini 3.5 Flash) APIを統合し、高精度なマークダウン解説文を自律生成してからDifyナレッジへアップロード・同期する仕組みを標準化しました。

## 7. 定量的RAG評価の CI/CD 常時自動テスト化

手動による評価のみであったRAGモデルに対し、定量的正解データセット (`tests/evaluation_dataset.json`) とLLM判定モデル (`gemini-2.5-flash`) を組み合わせた定量的評価自動テストを実行可能。回答品質を `Perfect` (+1.0), `Acceptable` (+0.5), `Missing` (0.0), `Incorrect` (-1.0) の4段階で定量評価し、平均スコアを計測。
本番・CI/CD環境における回帰テストとして機能し、評価結果レポート (Markdown) と履歴履歴 (CSV) を自動蓄積します。

## 8. Rustへの全面移植とTUIスクロール表示ハック

CLIおよびTUIコマンドを高速かつポータブルなRustに移行。TUI描画ライブラリ `ratatui` を使用した NotebookLM 風マルチビューワーを開発。
マルチバイト文字（日本語）を含む折返しが発生するチャット履歴ウィンドウにおいて、文字幅 (半角1, 全角2) と幅に応じた折り返し文字数をリアルタイムかつ厳密に計算する「カスタム行折返しバッファ計算ロジック」を実装し、1行単位で完璧に追従・スクロール可能なターミナルチャット操作を実現しました。
